Date: 2026-06-26 | Auditor: Claude Sonnet 4.6 | No code was modified

1. EXECUTIVE SUMMARY
================================================================================

The CTA system is structurally sound. The loader, registry, service, and node
all function correctly and all 104 automated tests pass. The greeting guard
works perfectly. The majority of well-formed, specific queries route correctly.

However, the audit identified one confirmed URL typo, five root-cause bugs
affecting at least 13 real query patterns, and two data gaps. The root causes
are concentrated in a small number of overly broad single-word aliases in
Courses (Learning, Training, Education), FAQ (Help), and About (ManaScience).
Because aliases carry weight=3 — higher than trigger examples (2) — a single
overly broad alias consistently beats a perfectly good trigger match in the
correct CTA.

Overall CTA Health Score: 68 / 100
(86.9% on the stress test, discounted further for the URL typo and the
severity of the alias pollution root cause)

--- Issue 2: Incorrect Category Prioritization (Therapy vs About) ---

Status: CONFIRMED BUG — Root Cause Identified

Failing query : "What therapies does ManaScience offer?"
Expected      : therapies/general
Actual        : about/about

Root cause: "ManaScience" is an alias for the About CTA (alias weight = 3).
The About CTA has no topic-scoping — it matches any question that contains
the word "ManaScience" anywhere. therapies/general has no alias list at all,
so its trigger examples (weight = 2) lose to the About alias (weight = 3).

Same root cause also affects:
  - "Is ManaScience free?"          → about/about (should be subscription)
  - "Is ManaScience GDPR compliant?" → about/about (should be privacy)
  - "Can ManaScience help my child?" → faq/faq tie (should be about/about)

Severity: CRITICAL — any question naming the platform gets hijacked.

Recommended Fix:
  Move "ManaScience" and "Mana Science" from the Aliases section to the
  Trigger Examples section in data/cta/about/about.md. As triggers (weight=2)
  they can still match, but a genuine trigger in another CTA will outrank them.


--- Issue 3: Missing Neuroplasticity CTA ---

Status: PARTIALLY CONFIRMED BUG — Depends on Topic Field

When the Understanding Node supplies a correct topic (Neuroplasticity, Brain
Rewiring, Brain Plasticity), the Neuroplasticity CTA is correctly returned.
Aliases and trigger examples are comprehensive.

Bug occurs in this narrow scenario: topic = "Brain Training", message = "Brain exercises."
  - search_text = "Brain exercises. Brain Training"
  - "training" (from Brain Training) matches "Training" alias on Courses (weight=3)
  - "Brain Training" matches Neuroplasticity related topic (weight=1)
  - Courses wins 3-1.

Root cause: The alias "Training" on courses/courses.md is a single generic word
that absorbs compound phrases containing "training" when they appear in topic.

Severity: HIGH — Neuroplasticity CTA is silently lost when topic contains
"Brain Training".

Recommended Fix:
  Remove "Training" as a single-word alias from data/cta/courses/courses.md.
  Alternatively, add "Brain Training" as an alias on the Neuroplasticity CTA,
  giving it weight 3 to beat Courses' topic weight of 1.

BUG-01 — "ManaScience" Alias Pollution
---------------------------------------
Severity : CRITICAL
File     : data/cta/about/about.md

Problem:
  "ManaScience" is in the Aliases section (weight=3). Any question that
  contains the word "ManaScience" routes to About regardless of intent.
  Alias weight (3) is higher than trigger weight (2), so any trigger match
  in a different CTA loses.

Confirmed failing queries:
  - "What therapies does ManaScience offer?" → about/about (should: therapies/general)
  - "Is ManaScience free?"                   → about/about (should: subscription)
  - "Is ManaScience GDPR compliant?"         → about/about (should: privacy)
  - "Can ManaScience help my child?"         → faq/faq tie  (should: about/about)

Fix:
  In data/cta/about/about.md, move "ManaScience" and "Mana Science" from the
  Aliases section to the Trigger Examples section. They keep matching, but
  other CTAs' specific triggers can now outrank them (trigger weight = 2).
  Also add explicit ManaScience-named triggers to Subscription, Privacy, and
  Therapy Library files (see BUG-02 fix details).


BUG-02 — "Learning" and "Training" Alias Pollution in Courses
--------------------------------------------------------------
Severity : HIGH
File     : data/cta/courses/courses.md
Aliases  : Learning (weight=3), Training (weight=3), Education (weight=3)

Problem:
  These three single generic words match in a vast number of non-course
  contexts. Because aliases weight 3, they beat any topic-level or
  trigger-level match in the correct CTA.

Confirmed failing queries:
  - "My child has learning difficulties."      → courses/courses (should: conditions/general)
  - "Learning difficulties."                   → courses/courses (should: conditions/general)
  - "Brain exercises." (topic: Brain Training) → courses/courses (should: neuroplasticity)

Secondary cause:
  conditions/general has "Learning Difficulty" (singular) as an alias, but
  "learning difficulties" (plural) fails to match it. The engine performs
  exact normalized matching only — no stemming.

Fix:
  1. Remove "Learning", "Training", "Education" as standalone single-word
     aliases from data/cta/courses/courses.md.
  2. Add "Learning Difficulties" (plural) as an alias to
     data/cta/conditions/general.md.

BUG-05 — "Autism" Related Topic False Positive
-----------------------------------------------
Severity : MEDIUM
Files    : data/cta/therapies/neurofeedback.md,
           data/cta/therapies/arrowsmith.md

Problem:
  "Autism" appears as a related topic on both Neurofeedback and Arrowsmith.
  When a user asks about autism and no Autism CTA exists, the query routes
  to therapies/neurofeedback via the related-topic match (weight=1). The
  tie-breaker (reverse alphabetical category sort) picks Therapy over
  Neuroplasticity.

Confirmed failing query:
  - "What is autism?" (topic: Autism) → therapies/neurofeedback (should: None)

Fix:
  Remove "Autism" from the Related Topics sections of neurofeedback.md and
  arrowsmith.md. The trigger examples "Can Neurofeedback help autism?" and
  "What conditions can Arrowsmith support?" already exist and are sufficient.
  The related topic is redundant and causes false positives.


BUG-06 — Substring Match Overfires on ADHD Trigger
----------------------------------------------------
Severity : MEDIUM
File     : data/cta/conditions/adhd.md

Problem:
  The ADHD trigger example "My child struggles in school because of ADHD."
  (normalized: "my child struggles in school because of adhd") is matched
  by the engine's text_norm-in-phrase_norm substring check against the
  shorter user message "My child struggles in school." — because the user
  text is a substring of the trigger phrase.

  Both ADHD and conditions/general score 2 for this message, but since ADHD
  is a non-Library type and conditions/general is Library, ADHD wins within
  the Condition category.

Confirmed failing query:
  - "My child struggles in school." → conditions/adhd (should: conditions/general)

Fix:
  Rewrite the ADHD trigger to require an explicit ADHD signal, for example:
    Old: "My child struggles in school because of ADHD."
    New: "My child with ADHD struggles in school."
  The existing trigger "My child can't focus in school." (without ADHD
  mention) is a better ADHD-specific trigger and should be retained.

NEW-05 — "M.N.R.I." Alias Normalization Risk
---------------------------------------------
Severity : LOW
File     : data/cta/therapies/mnri.md

Problem:
  The alias "M.N.R.I." normalizes to "m n r i" — four single-character
  tokens. The token-subset matching rule means any text containing all four
  letters m, n, r, i as individual tokens would technically match. In
  practice this is unlikely, but it is a latent fragility with no benefit
  since "MNRI" already covers this case.

Fix:
  Remove "M.N.R.I." from the Aliases section of data/cta/therapies/mnri.md.
  The "MNRI" alias covers all practical cases.

11. SUGGESTED IMPROVEMENTS
================================================================================

1. Alias promotion discipline
   Aliases should only contain proper nouns, acronyms, and brand names
   (MNRI, FAQ, Community Hub). Generic English words (Learning, Training,
   Help, Support, Reading) belong in trigger examples, not aliases.

2. Alias scope guard
   Before adding a new alias, verify it does not appear as a single token
   inside any trigger example or natural language phrase from a different
   category's expected queries.

3. Parent-language trigger parity
   Mirror every "My child..." trigger with "My son..." and "My daughter..."
   variants, or add "son" and "daughter" as related topics on conditions/general.

4. Add ManaScience compound triggers per category
   For each category that might receive platform-name queries, add explicit
   compound triggers:
     - "What therapies does ManaScience offer?" → therapies/general
     - "Is ManaScience free?"                   → subscription
     - "Is ManaScience GDPR compliant?"         → privacy

5. Plural/singular coverage
   Audit all aliases that are single nouns for their plural forms:
     - "Learning Difficulty" → add "Learning Difficulties"
     - Check all condition aliases have both singular and plural forms
